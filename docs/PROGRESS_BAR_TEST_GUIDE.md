# 回测进度条功能手动测试指南

## 功能说明

在回测仿真开始后，会显示一个实时更新的进度条，避免长时间无响应，提升用户体验。

## 测试步骤

### 1. 启动Web服务

```bash
cd /path/to/stocks
python -m gui.web
```

服务将在 http://127.0.0.1:5000 启动

### 2. 访问首页并选择股票

1. 浏览器访问 http://127.0.0.1:5000
2. 在搜索框输入股票代码或名称，如 `600900` 或 `长江电力`
3. 点击"搜索"按钮
4. 点击"选择此股票"按钮

### 3. 选择策略

1. 在策略选择页面，点击任一策略卡片，如"SMA 策略"
2. 页面跳转到运行模式选择

### 4. 选择运行模式

1. 点击"回测仿真"卡片
2. 页面跳转到策略配置页面

### 5. 配置策略参数并提交

1. 填写起始日期和结束日期，如 `20230101` 到 `20231231`
2. 配置其他参数（或使用默认值）
3. 点击"开始回测"按钮

### 6. 观察进度条

页面会跳转到进度展示页面，应该能看到：

**初始状态（0%）：**
- 页面标题：回测仿真进行中
- 状态提示：正在执行回测...（带旋转动画）
- 进度条：0%，紫色渐变背景
- 进度文字：准备开始...

**运行中（1-99%）：**
- 状态提示：正在执行回测...（带旋转动画）
- 进度条：实时更新，平滑过渡
- 进度文字：正在处理第 X / Y 天的数据
- 百分比：实时显示，如 45%

**完成状态（100%）：**
- 状态提示：✓ 回测完成！（绿色背景）
- 进度条：100%
- 进度文字：回测完成！
- 按钮：查看回测结果（显示并可点击）

**错误状态（如发生错误）：**
- 状态提示：✗ 回测失败：错误信息（红色背景）
- 进度条：停在错误时的百分比

### 7. 查看结果

1. 点击"查看回测结果"按钮
2. 页面跳转到回测结果展示页面
3. 可以看到详细的回测数据、图表和交易明细

## 预期效果

1. **实时反馈**：进度条应该实时更新，不会卡在某个百分比很久
2. **流畅动画**：进度条过渡平滑，有渐变色效果
3. **清晰状态**：不同状态有明显的视觉区分（蓝色运行中、绿色完成、红色错误）
4. **无卡顿**：整个过程中浏览器不会出现"页面无响应"的提示
5. **准确计数**：进度文字显示的处理天数应该与实际数据量匹配

## 技术实现

- **后端**：使用 Server-Sent Events (SSE) 推送进度更新
- **前端**：使用 EventSource API 接收进度，实时更新DOM
- **回测**：在后台线程执行，通过回调函数报告进度
- **进度计算**：基于数据行数计算百分比（当前索引 / 总行数 * 100）

## 注意事项

1. 数据量小的回测（如1个月）可能完成很快，进度条会快速跳过
2. 数据量大的回测（如1年以上）能更好地观察进度更新
3. 首次运行可能需要下载数据，会有额外的等待时间（但这部分不显示进度）
4. 如果浏览器禁用了SSE，进度条将无法更新（但不影响回测执行）
